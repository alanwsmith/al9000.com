== extends "/templates.inc/pages/javascript.html" == block toml title = "A Media Source Extensions (MSE) Video Player" created = 2026-04-06T19:30:12-04:00 status = "draft" == endblock == block main == filter md ## Overview - This is a very basic approach to using the Media Source API (aka [Media Source Extension](https://developer.mozilla.org/en-US/docs/Web/API/Media_Source_Extensions_API)) to load videos in a <video> tag instead of using a `src` attribute that points to a single file (or [HLS](https://en.wikipedia.org/wiki/HTTP_Live_Streaming) playlist). - This is primarily a prototype I'm working on for my [Multi-Player project](/multi-player/). Chrome slows way down as the number of videos on the page increases. Hopefully, this approach can keep things snappy. - This is also prep work for making a version that works with encrypted video files (e.g. by [Password Protecting Static Site Content with WASM](/rust/wasm/static-site-file-decryption/)). - You should probably use [VideoJS](https://videojs.org/) instead of this in most cases. - This does not currently check to see if the mime type is supported. I'm using [.webm](https://www.webmproject.org/) in the examples and have things lined up already with the videos (i.e. `video/webm; codecs="vp9, vorbis"`). See [the doc examples](https://developer.mozilla.org/en-US/docs/Web/API/SourceBuffer#examples) for details on how to check in situations where you can't control the input files. - My original version of this didn't use global variables. I ran into an issue where missing audio was breaking chrome and added the feature to track videos by UUID in the global `videos` variable during the bug hunt. I'm fine with it being there. ## The Video Here's the output demo video to get started. It's from [pexels](https://www.pexels.com/video/a-drone-shot-of-a-church-in-belalp-3818213/) which is a nice place to get free videos. == endfilter == include parent(file_path) + "/page.html" == filter md ## The JavaScript The main JavaScript consists of a variable and two functions in the module shown here: == endfilter == filter highlight_block("js", "media-source-player.js") == include parent(file_path) + "/media-source-player.js" == endfilter == filter md - The `loadVideo` function takes a selector that points to an existing `video` element and a config. - The format of the config is in the HTML below. - The config includes a fallback URL as well as the links to the individual parts that the MediaSource uses. - A check is made to verify the MediaSource loaded. If not, the fallback URL is used. - I'm using a direct link to a `.webm` file for the fallback. In practice, I'd probably use an [HLS](https://en.wikipedia.org/wiki/HTTP_Live_Streaming) playlist. - The module handles adds an event listener to the video element for [canplaythrough](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/canplaythrough_event). It's fired when the browser estimates it has enough data to play the video without having to stop for buffering. - I left the `console.log('processing segment...'` line in to show how the `canplaythrough` event fires while the individual segment files are still being loaded. ## The HTML Here's the HTML that goes on the page: == endfilter == filter highlight_block("html") == include parent(file_path) + "/page.html" == endfilter == filter md - The video tag has an `id` attribute that's used in the selector to grab the element. - The `config` includes the `codec` for the video in addition to the various paths. Fair warning that there's a deep rabbit hole with video codecs. ## The Prep Script The sample video I used didn't have an audio track with it. It worked fine in Safari and Firefox. It broke in chrome. It took a while to figure out that the missing audio was the problem. This is the `ffmpeg` command I used to fix the issue.
