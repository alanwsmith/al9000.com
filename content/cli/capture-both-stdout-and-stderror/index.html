Terminals pick up the content from the streams and print it out for you to see. What's not always clear is that there are two streams in play: - `STDOUT` - The default output when everything works as expected. - `STDERR` - For when things to wrong and errors occur. If you delete an existing file with: == filter highlight_command() rm -v file-that-exists.txt == endfilter It outputs: == filter highlight_text() file-that-exists.txt == endfilter The `-v` flag sets the `rm` command to `verbose` mode which prints out file names as they are removed. The `STDOUT` stream delivers the names. If you run the same command with a path that doesn't exist you'll get a different message: == filter highlight_command() rm -v path-that-does-not-exist.txt == endfilter Outputs: == filter highlight_text() rm: path-that-does-not-exist.txt: No such file or directory == endfilter There's a behind-the-scenes difference. `rm` uses `STDOUT` for the original message and `STDERR` for the message about the missing file. ## Visibility and Pipes If you're using a terminal to run commands you won't see a difference between `STDOUT` and `STDERR`. Terminals combine them automatically. Things change when you pass the output of one command into another via a pipe (i.e. `|`). The `STDOUT` streams gets piped through. The `STDERR` stream does not.
